# 자바스크립트란 어떤 녀석인가.

객체지향 프로그래밍을 지원 하는 인터프리터 방식.
  컴파일 되지 않고 인터프리터 된다고 스트립트 언어는 아니다.
  요놈도 다양한 기능을 갖춘 완벽한 프로그램 언어임.

변수의 타입을 명시 할 필요가 없는 타입 제약이 약한 언어
이 녀석의 객체는 프로퍼티 이름을 임의 프로퍼티 값에 연결 시킴.
이 녀석이 가진 객체 지향 상속 메커니즘은 프로토타입(prototype) 기반.

기본 데이터 타입은 숫자, 문자열, 불리언
기본 객체는 배열, 날짜, 정규 표현식 등.

웹 서버가 아닌 클라이언트 컴퓨터에 의 해 실행 된다고, 클라이언트 측 자바스크립트라고 함.
그 예가 웹브라우저에 내장된 자바스크립트가 HTML웹페이지 내에 포함된 스크립트를 실행.

코어 자바스크립트 > 클라이언트 자바스크립트
뭔가 자바스크립트의 쓰임새가 많고 그 중에 가장 유명한것이 클라이언트 자바스크립트 인듯.

# 버전 이야기

ECMA에 의 해 표준화 됨.( ECMA-262 = ECMAScript )
모질라 재단, 마이크로소프트 의 인터프리터가 이 표준을 구현함.

이 외에도 ECMA-357( E4X - ECMASctipt for XML )
XML데이터 타입 지원과 값을 조작 하기 위한 연산자 등을 추가 한것.

ECMA-262 4번째 버전이 검토중 . ( 이거 지금 이야기 맞음? 이게 ES6 이야기인듯. )
여기에는 타입 제약을 강화 하고, 클래스 기반 상속 지원 등이 포함 됨.




# 클라이언트 측 자바스크립트
이 것은 자바스크립트 인터프리터가 웹브라우저에 내장 된 것.
자바스크립트 인터프리터의 스크립트 지원 + 웹브라저에 의해 정의 되는 문서 객체 모델 (DOM)
정적인 웹컨텐츠에 동적인 성질을 부여 함.
스크립트로 HTML 문서 내용에 접근 할 수 있다는 것.

# 다른 환경에서의 자바스크립트는?
이 녀석은 여기저기 쓸수 있다. 인터프리터만 있다면.
넷스케이프, 마이크로소프트에서 자바스크립트 인터프리터를 공개 해왔음

# 자바스크립트를 테스트 해보는 방법
간단해. 별거 필요없어.
텍스트 편집기 + 웹브라우저만 있음 뭐든 해볼수 있음.
더 간단하게 하고 싶다면 웹브라우저 주소창에 javascript:블라블라 라고 세미콜론으로 구분해서 치면 결과를 보여줘.
이 방법은 javascript URL 모조 프로토콜 ( pseudo protocol ) 이라고 하는데,
네트워크 상의 리소스 접근을 위해 쓰는 http, https 같은 프로토콜이랑은 다른게
그냥 단순히 자바스크립트의 결과를 출력 하는데 쓰는 가짜 프로토콜이야.

그리고 웹브라우저의 콘솔에서도 가능해.

뭔가 의도대로 되지 않을때엔 디버깅을 해보고 싶을 껀데 변수에 값을 출력하는 코드를 집어 넣고 뭔 일이 벌어지고 있나 보면 되. 그떄 document.write(), alert()같은 메소드로 출력 해보면 알수 있어.
for/in루프 + alert()메소드를 사용하면 객체의 모든 프로퍼티들의 이름과 값을 출력 할 수 있어.

jslint 라는것도 있고.
이미 우린 크롬 콘솔을 쓰고 있지만. 그렇다고 하네.



# 기본적인 어휘구조 ( lexical structure )
- Unicode

- 대소문자 구분해야 함.

- 줄바꿈과 세미콜론.
  - 줄 바꿈 하면 문장을 구분 해주긴 하는데 세미콜론 생략 안하는게 좋음.
    줄바꿈 하면 의도치 않은 상황이 발생 할수 있음.
  - 토큰 - 세미콜론, 이스케이프 시퀀스, 공백 같은 구분 지을때 사용하는 구분자를 통칭.
- 주석이야 알지.

- 리터럴 ( literal )
  - 프로그램에 직접 나타나는 데이터 값. 그냥 데이터는 리터럴이라고 표현하면 되는가?
  - 객체 리터럴을 위한 표현식을 지원
    { x:1, y:2 } // 객체 초기자
    [1,2,3,4] // 배열 초기자

- 식별자
  - 변수나 함수, 루프문의 레이블의 이름이름이름.
  - 첫문자는 알파벳, 언더바, 달러. 숫자는 허용하지 않음.
  - Unicode 이스케이프 시퀀스 사용할수 있음.
  - 예약어는 사용할 수 없음.

- 예약어
  - 예약되어 있는 키워드.
  - 자바스크립트가 이거 내가 쓸거임 ㅇㅇ 하고 찜 해 놓은 단어들인듯.
  - 그 자체가 자바스크립트 언어의 문법을 구성하는 요소.
  - 이것 뿐만 아니라 미리 정의 된 전역 변수나, 함수 이름도 사용하지 말것.



# 데이터 타입과 값

- 숫자
  - 정수, 실수 구별 하지 않음. 모두 실수로 표현
  - 정수 리터럴
    - 10진수: 숫자 시퀀스, 즉 연속된 숫자 형태로 작성.
    - 16진수, 8진수: 8진수가 0으로 시작 하기 때문에 정수 리터럴을 0으로 시작하게 작성하지 마시라. 10진수 8진수 뭘로 인식 할지 알수 없다.
  - 부동소수점 리터럴
    - 지수표기법...
  - 산술연산자, 수리 함수 제공 그거슨 Math라는 객체로 제공됨.
  - 특수한 숫자 값: infinity, NaN



- 문자열
  - Unicode 문자, 숫자, 문장부호를 연속된 형태로 표현.

  - 문자열 리터럴
    - 줄바꿈시에는 /n 사용.
    - 생략부호 살릴때는 역슬래시
    - 작은 따옴표, 큰 따옴표로 문자열 리터럴을 작성. HTML, 자바스크립트 섞어 쓸땐 따옴표 교통정리 해서 쓰라.

  - 이스케이프 시퀀스
    - 역슬래시로 시작하는. 일반적인 해석방식을 벗어나 특수한 방식으로 해석하게 해주는 놈.
    - 문자 맨 앞에 오는 역슬래시는 무시된다.

  - 문자열 조작
    - + 하면 더해짐.
    - length 쓰면 갯수 구할 수 있음
    - 문자열의 마지막 문자 구하기: a.charAt(index)
    - 특정 인덱스의 문자 추출: sunstring(1, 4)
    - 특정 문자 위치 찾기: indexOf('a')
    - 배열 표기법 사용: a[s.length-1] = a.charAt(index)

  - 숫자 -> 문자열로 변환
    - 숫자 + 빈 문자열: 100 + "";
    - String(): var foo = String(nunber);
    - toString(): foo = nunmer.toString();
      - 기수를 전달인자로 넘기면 맞게 변환 가능. 생략하면 10진법
    - toFixed(): 소수점 이하 숫자를 지정된 개수 만큼 출력 ( 지수표기법x )
    - toExponential(): 소수점 앞의 숫자하나 + 지정된 개수의 소수점 이후 숫자 ( 지수표기법o )
    - toPrecision(): 지정된 수의 유효숫자 개수 만큼 출력

  - 문자열 -> 숫자로 변환
    - 문자열이 숫자 문맥을 타면 숫자로 변환이 되지만 명시적이지 않아 별로 인듯.
      - "21" + "10" //31출력
      - var foo = string - 0; // 문자열에서 0이 빠짐.
    - Number() 타입 선언.
      - 10진수만 제대로 변환
      - 숫자 다음에 스페이스 외의 문자 안됨.
    - parseInt() / parseFloat() 메소드
      - 어떤 형태의 숫자라도 변환, 반환 가능
      - 숫자 다음에 나오는 숫자가 아닌 문자는 모두 무시
      - parseInt() 정수 변환, 기수를 전달인자로 넘길수 있음
      - parseFloat() 정수 & 부동소수점
      - 변환 못하면 NaN


- 불리언
  - true / false
  - 제어구조 내에 사용
  - 숫자 문맥 -> true : 1 false : 0
  - 문자열 문맥 -> 문자열 "true" "false"

  - 숫자 -> 0, NaN : false
  - 문자 -> 빈문자열 : false
  - null , undefined : false

  - 명시적 타입 변환
    - Boolean()
    - !!

- 함수
  - 미리 정의 되어 있는 실행 가능한 코드
  - 한번 정의 하면 여러번 실행 가능.
  - 전달인자 agument / 매개변수 parmeter 를 넘겨 받고 결과를 반환.
  - 자바스크립트에서 Math 함수 같은거 제공해.
  - 자바스트립트에서 함수는 코드로 조작할 수 있는 형태의 값. 유연성b.
    - 변수, 객체 저장 가능
    - 다른 함수의 전달인자로 넘겨 줄수 있음.
  - 값 (literal)이므로 객체 프로퍼티에 할당 될수 있다.
    - 어떤 객체의 프로퍼티로 할당되면 = 메서드 method


  - 함수 리터럴 문법 ( Lambda function )
    - function 키워드 / 함수 이름(생략가능)
    - function foo(x) { return x+x }
    - var foo = function(x) { return x+x }


- null / undefined




- 객체 ( object )
  - 이름 붙은 값들의 모임
  - 복합 데이터 타입. 값들의 모임. 데이터들의 모임.
    - 값 = 프로퍼티
    - 객체의 프로퍼티 참조 방법 : imgae.width

  - 종류
    - 이름 붙은 값들의 순서 없는 모임
    - 번호가 붙은 순서 있는 값들의 모임( 배열 )
    - 함수: 특정 동작을 수행 하기 위해 호출
    - Date, RegExp Error
